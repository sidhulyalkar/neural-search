# Architecture Connectivity Audit — 2026-07-01

**Purpose:** Determine which of the repository's implemented KG/graph builder modules actually reach the graph the search API serves from, versus which are isolated. Triggered by discovering (earlier this session) that the Methodology Registry work was built into a dev/CLI-only pipeline rather than production, and that `scripts/build_real_corpus_graph.py` — the actual production builder — never called it.

## The three build pipelines

| Pipeline | Entry point | Output | Status |
|---|---|---|---|
| **Production search graph** | `scripts/build_real_corpus_graph.py::build_graph()` | `data/graph/neural_search_graph.real_corpus.json` (7,680 nodes / 141,218 edges as of today) | **The only graph the search API loads at runtime.** |
| **Composed conceptual layers** | `neural_search/ingestion/compose_kg.py` | `artifacts/kg/composed_kg.jsonl` | Built, tested, but **not read by anything at search time** except three specific side-channel loaders (below). |
| **Dev/CLI record graph** | `neural_search/graph/builder.py::build_graph_from_records()` | in-memory / CLI output via `neural_search/graph/build_graph.py` | Used by tests and `build_graph.py` CLI only, not production. |

## What this means

1. **This was a systemic pattern, not a one-off bug.** The methods_builder gap found 2026-07-01 was the same shape as 7 other builders (disorder, allen_connectivity, paradigm, oscillation, species_homology, hcp_connectivity, scholarpedia, concept), all now reconnected — 8/8 originally-identified orphaned KG-merge builders live in production as of 2026-07-02.
2. **Three side-channel loaders (neurosynth, NER, citations) prove a *second*, equally valid reconnection pattern**: bypass the composed graph and read a purpose-built artifact directly (the same technique `reanalysis_bridge_builder.py` used for `ner_kg.jsonl`). All three are genuinely connected this way, not dead — the follow-up audit's job was verifying that claim, not assuming it, and one real fragility (`neurosynth_builder.py`'s dependency on an unregenerated file) was found and fixed as a result.
3. **`citation_builder.py` is the one deliberately-deferred case** — its full reconnection is blocked on a real prerequisite (paper nodes in the production graph) rather than pure wiring, so forcing it in now would have exceeded the scope of a wiring fix.

## Recommendation

The KG-builder reconnection sweep is complete: every builder module in `neural_search/ingestion/` that produces graph-like output is now either (a) merged directly into production, (b) genuinely reachable via a documented side-channel artifact, (c) confirmed dead and removed, or (d) explicitly deferred with a stated blocking reason (`citation_builder.py`). No remaining silent orphans.
